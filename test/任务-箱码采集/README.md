# 箱码采集任务

## 任务描述

本任务实现箱码数据的采集、处理和PLC写入功能，主要包括以下步骤：

1. **启动TCP服务端**：监听端口8888，等待箱码数据传入
2. **接收箱码数据**：接收并存储箱码数据
3. **查询数据库**：查询装箱表数据
4. **数据验证**：检查数据库是否有数据，无数据则报警
5. **写入PLC**：将序号写入PLC寄存器D5502

## 执行规则

### 定时任务执行逻辑
- 必须按步骤从流程中一步一步执行
- 如果第一步、第二步没有数据流转下去，就一直等，且不用能开新任务，
- 获取到数据后如果不满足条件，记录日志并返回数据获取，只有满足条件才继续流转，不然一直等待。
- 只有有数据且能流转才继续执行下去

### 监听事务类任务执行逻辑
- 等待第一步的数据传入，然后进行流转
- 如果一直没有数据传入，就一直等待
- 数据传入后，按照流程步骤依次执行

## 环境准备

### 1. 数据库配置
- 创建MySQL数据库：`iot_db`
- 创建装箱表：
  ```sql
  CREATE TABLE 装箱表 (
    产品序列号 VARCHAR(50),
    箱号 VARCHAR(50),
    序号 INT
  );
  ```
- 插入测试数据：
  ```sql
  INSERT INTO 装箱表 VALUES 
  ('04001EM01008082406530276', 'BOX001', 1),
  ('04001EM01008082406530277', 'BOX001', 2),
  ('04001EM01008082406530578', 'BOX001', 3),
  ('04001EM01008082406530875', 'BOX001', 4);
  ```

### 2. PLC配置
- PLC类型：汇川H5U-A8S
- IP：127.0.0.1
- 端口：502
- 寄存器：
  - D5501：用于接收箱码
  - D5502：用于接收序号

#### 设计器中的 PLC 节点
- 寄存器地址填写规则：客户给 `D5502` 时，在 PLC 节点里填写数值 `5502`。
- 从站号 `unitId`：通常保持默认 `1` 即可（当前表单不要求填写）。
- 在流程设计器左侧 **设备与通信** 中可拖拽 **PLC 读取**、**PLC 写入**。
- **PLC 写入**：在节点表单中配置 `registers` 列表，可添加多条（多地址写入）；与后端 `PLC_WRITE` 一致。
- **PLC 读取**：配置 `reads` 列表（每段起始地址 + 连续寄存器个数），结果默认变量 `plcReadResult`。

#### PLC_WRITE 配置建议（字符串写入）
- 对箱码（字符串）建议在对应寄存器项设置：`"writeAs": "STRING_ASCII"`。
- 对序号（数字）建议设置：`"writeAs": "INT"`（或留空使用 `AUTO`）。
- `AUTO` 模式下：数字走单寄存器写入，非数字字符串自动拆分为多个连续寄存器写入。

### 3. 网络配置
- TCP服务端端口：8888
- TCP客户端：使用网络调试助手发送箱码数据

## 测试方法

1. 启动PLC模拟器
2. 启动箱码采集流程
3. 使用网络调试助手连接到TCP服务端（127.0.0.1:8888）
4. 发送箱码数据，如 `04001EM01008082406530276`
5. 查看系统日志，验证流程执行情况
6. 检查PLC寄存器D5502的值
