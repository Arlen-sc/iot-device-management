# 箱码采集测试运行指南

## 测试环境准备

### 1. 网络环境
- 确保所有设备在同一局域网内
- IP地址分配：
  - PLC模拟器：127.0.0.1:502
  - TCP客户端：127.0.0.1
  - 数据库：127.0.0.1:3306

### 2. 设备准备

#### PLC模拟器
- 模拟汇川H5U-A8S
- 监听端口：502
- 支持Modbus TCP协议
- 寄存器D5502用于接收序号

#### 数据库
- 创建MySQL数据库：`iot_db`
- 创建装箱表：
  ```sql
  CREATE TABLE 装箱表 (
    产品序列号 VARCHAR(50),
    箱号 VARCHAR(50),
    序号 INT
  );
  ```
- 插入测试数据：
  ```sql
  INSERT INTO 装箱表 VALUES 
  ('04001EM01008082406530276', 'BOX001', 1),
  ('04001EM01008082406530277', 'BOX001', 2),
  ('04001EM01008082406530578', 'BOX001', 3),
  ('04001EM01008082406530875', 'BOX001', 4);
  ```

#### TCP客户端
- 使用网络调试助手
- 连接到TCP服务端：127.0.0.1:8888
- 发送箱码数据：如 `04001EM01008082406530276`

### 3. 系统配置

#### 步骤1：导入流程配置
1. 打开系统Web界面
2. 进入「任务管理」→「流程设计」
3. 点击「导入流程」
4. 选择文件：`test/任务-箱码采集/箱码采集流程.json`
5. 保存流程配置

#### 步骤2：配置数据库连接
1. 进入「数据源管理」
2. 添加MySQL数据源：
   - 主机：127.0.0.1
   - 端口：3306
   - 数据库：iot_db
   - 用户名：root
   - 密码：your_password

## 测试执行步骤

### 1. 启动箱码采集流程
1. 进入「任务管理」→「任务列表」
2. 找到导入的「箱码采集流程」
3. 点击「启动」按钮

### 2. 验证流程执行

#### 步骤1：启动TCP服务端
- 确认TCP服务端成功启动，监听端口8888

#### 步骤2：接收箱码数据
- 使用网络调试助手连接到TCP服务端
- 发送箱码数据，如 `04001EM01008082406530276`
- 确认系统接收到数据

#### 步骤3：查询数据库
- 确认系统查询装箱表数据
- 验证数据查询结果

#### 步骤4：检查数据库数据
- 如果数据库无数据，确认系统报警
- 如果数据库有数据，确认系统继续执行

#### 步骤5：写入PLC寄存器
- 确认系统将箱码写入PLC寄存器D5501
- 确认系统将序号写入PLC寄存器D5502
- 使用PLC模拟器验证寄存器值

## 执行规则验证

### 定时任务执行逻辑
- **测试场景1**：TCP服务端未接收到数据
  - 预期：流程跳过并完成
- **测试场景2**：数据库无数据
  - 预期：流程跳过并完成，同时报警
- **测试场景3**：数据正常流转
  - 预期：流程完整执行

### 监听事务类任务执行逻辑
- **测试场景**：启动流程后等待数据
  - 预期：流程一直等待，直到接收到数据后继续执行

## 异常处理

### 1. 网络异常
- 模拟TCP服务端连接失败
- 查看系统日志，确认异常处理

### 2. 数据库异常
- 模拟数据库连接失败
- 查看系统日志，确认异常处理

### 3. PLC异常
- 模拟PLC连接失败
- 查看系统日志，确认异常处理

### 4. 数据异常
- 发送格式错误的箱码数据
- 查看系统日志，确认异常处理

## 测试结果验证

### 1. 日志验证
- 检查系统日志，确认各步骤执行情况
- 确认无错误日志

### 2. 网络验证
- 使用网络调试助手发送箱码数据
- 确认数据传输正常

### 3. 数据库验证
- 检查数据库查询结果
- 确认数据正确

### 4. PLC验证
- 检查PLC寄存器D5502的值
- 确认值与数据库中的序号一致

## 测试完成

### 1. 停止任务
- 停止运行中的箱码采集流程
- 关闭模拟设备

### 2. 生成测试报告
- 记录测试过程中的所有操作
- 记录测试结果和异常情况
- 生成完整的测试报告

## 注意事项

1. 测试前确保PLC模拟器正常运行
2. 准备箱码数据用于测试
3. 记录测试过程中的异常情况
4. 测试完成后生成测试报告
5. 保存测试配置，以便下次测试使用
